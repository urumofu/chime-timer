# Game OCR Tool 連携仕様（開発者用メモ）

※本ドキュメントは、時報タイマーを `Game OCR Tool` (main.py) と連動させる際の個人的な仕様メモです。配布用の仕様書とは分離しています。

## 1. 連動時のトリガーと起動フロー
- **呼び出し元**: `Game OCR Tool` 画面右下に配置された「🔔」ボタン。
- **実行メソッド**: `main.py` 内の `open_chime_timer` 関数。
- **処理内容**:
  1. `socket` を用いて localhost のポート `8765` を簡易チェックし、サーバーが既に起動しているか（ポートが使用されているか）を確認します。
  2. サーバーが起動していない場合のみ、`subprocess.Popen` を使用して裏で `python -m http.server 8765` を開始します。
     - **Windows向け対策**: `creationflags=subprocess.CREATE_NO_WINDOW` を指定することで、余分なコマンドプロンプトの黒窓が出ないようにしています。
  3. `root.after(800, ...)` でサーバーの立ち上がりを少し（800ms）待ってから、既定のブラウザで `http://localhost:8765` を開きます。
  4. サーバーがすでに起動済みの場合は再度サーバーを立ち上げる処理は行わず、ブラウザを開く動作のみを行います。

## 2. 制限事項
- タイマーのバックグラウンドプロセスは、あくまでも Python 標準の `http.server` に依存しているため、Game OCR Tool 本体を終了しても Python のプロセスとして裏側に残り続ける仕様です。
- ポートの競合などを防ぐため、次回 Game OCR起動時に再度呼び出しても、「起動済みのためブラウザを開くだけ」という安全な分岐に入ります。
